无服务器架构 (Serverless) - 开始

简介：随着手机APP开发的兴起，BaaS技术解决方案也开始兴起，它们希望帮助APP开发者专注于APP开发，而简化后台的编写。开发者只需要使用它们的SDK，就能迅速的完成社交网络账户整合，消息推送，信息储存等功能。而AWS推出的lambda服务则更是BaaS在云平台实现的核心。

著名的BaaS提供商有已经被facebook收购的parse，开源的usergrid；

而AWS推出的lambda服务则更具有灵活性，因为它并不仅仅适用于APP开发者，而且可以根据公司的业务流程需要，整合不同的服务，简化系统架构。比如S3，DynamoDB，Cloudwatch，API Gateway等等都可以触发lambda的运行。

而且lambda服务不需要用户维护服务器，lambda根据请求数等计费，而且自动扩张以应付高请求数。对于用户来讲，它是一个Serverless的架构，因为用户并不需要租用维护服务器。有人也称之为FaaS（Function-as-a-Service），因为用户只需要写一个个计算单元。

现在各大云服务商也都有提供类似的Serverless架构：谷歌的 “Google Cloud Functions”，微软的"Microsoft Azure Functions"等等

Lambda 计算单元 （Lambda Function)

Lambda计算单元有三个参数event，context，callback；

Event根据来源不同有不同的格式。具体的格式请参考aws-event-mocks。 Context包含各种Lambda本身的信息。Callback负责返回。

  • event json
{
  "key3": "value3",
  "key2": "value2",
  "key1": "value1"
}
  • lambda function
console.log('Loading function');

exports.handler = function(event, context, callback) {
    //console.log('Received event:', JSON.stringify(event, null, 2));
    console.log('value1 =', event.key1);
    console.log('value2 =', event.key2);
    console.log('value3 =', event.key3);
    callback(null, event.key1); // Echo back the first key value

};

工具

现在已经有很多工具来帮助用户写、发布lambda计算单元。比如serverless, claudiajs, stdlib等等。

serverless新的版本v1已经改进，简化了很多，值得一用。

用户通常只需要通过serverless.yml文件来管理AWS服务同lambda计算单元之间的关系。

  • 管理S3

当S3有新文件或改动时，就会运行resize.js中的handle功能。具体请看s3

functions:
  resize:
    handler: resize.handler
    events:
      - s3: photos
  • 管理API Gateway

当API Gateway通过hello收到get是，会触发handler.js中的hello功能。具体请看api gateway

functions:
  index:
    handler: handler.hello
    events:
      - http: GET hello
