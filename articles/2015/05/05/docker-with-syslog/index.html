Docker with runit, part 2 rsyslog

May 5, 2015

# docker # runit # syslog

Before we start to use syslog within docker instance, I found some posts discussing docker zombie process problem and whether we should use ssh or just docker exec 11 Post series from phusion

docker exec is introduced since docker 1.4, user will get root access to the docker instance after they login. So, SSH is helpful in certain conditions. Rightscale shared their idea on "how to use SSH in docker with host SSH agent socket and how to handle logs", which is also worth reading.

Now, let's come back to rsyslog.

Run rsyslog

A syslog daemon is necessary for many services include kernel itself. We will still use runit to indicate how to use syslog within docker instance.

  • Runit script to run rsyslog, copy to /etc/service/rsyslog/run
#!/bin/bash
exec 2>&1
cd /
chgrp syslog /var/log
chmod g+w /var/log
exec rsyslogd -n
  • Dockerfile
#Dockerfile
FROM debian:jessie

# install, config rsyslog
RUN apt-get update && apt-get install rsyslog
RUN mkdir -p /etc/service/rsyslog/
COPY ./service/rsyslog/run /etc/service/rsyslog/run
RUN chmod 755  /etc/service/rsyslog/run

# install, config runit
RUN touch /etc/inittab
RUN apt-get update && DEBIAN_FRONTEND=noninteractive apt-get install -y -q runit

COPY runit_bootstrap /usr/sbin/runit_bootstrap
RUN chmod 755 /usr/sbin/runit_bootstrap

ENTRYPOINT ["/usr/sbin/runit_bootstrap"]
  • Build image, tag and run it
docker build .
docker tag 5416c3b71823 debian-runit:rsyslog

# --name:   set docker NAME, should be used before option -t
# -h:       set docker instance hostname
docker run -d --name runit-rsyslog1 -h runit-rsyslog1 -t debian-runit:rsyslog
  • Check

Check processes running in docker instance.

# docker exec -i -t runit-rsyslog1 bash
# pstree
runsvdir---runsv---rsyslogd-+-{in:imuxsock}
                            `-{rs:main Q:Reg}`
