Docker workflow, part 3: php app

May 8, 2015

# docker # php # wordpress # continuous integration # workflow

Many of our applications are written in php language. So, I'm going to dockerize our php applications in this post.

Start

Docker provides us with a compose and wordpress page. But the docker image orchardup/php5 is built from ubuntu 13.10 without php5-fpm package installed. OMG, you'd be kidding me.

Thus, I have to build the nginx image and php5-fpm myself. There are several options to choose:

Option 1: mount config volumes later when I start a container.
Option 2: build configs within docker image

And to enable the web container and the app container to talk with each other, there are two approaches: The first, two containers share a volume contains php5-fpm socket file. The second, use docker link.

To simplify my task, I would choose "build configs within docker image" and share a volume contains php5-fpm socket file between the web container and the app container.

Build docker image for web server

Dockerfile:

FROM ubuntu:14.04

RUN apt-get update && apt-get install -y software-properties-common

RUN \
  add-apt-repository -y ppa:nginx/stable && \
  apt-get update && \
  apt-get install -y nginx && \
  rm -rf /var/lib/apt/lists/* && \
  echo "\ndaemon off;" >> /etc/nginx/nginx.conf && \
  chown -R www-data:www-data /var/lib/nginx

# Clean up APT
RUN apt-get clean && rm -rf /var/lib/apt/lists/* /tmp/* /var/tmp/*


COPY nginx.conf /etc/nginx/sites-enabled/default

EXPOSE 80
EXPOSE 443

CMD ["nginx"]

Build the nginx image:

docker build -f Dockerfile.nginx -t "gankkank/ubuntu-trusty-nginx:latest" .

Build docker image for app server

Dockerfile:

FROM ubuntu:14.04
# 
RUN apt-get update && apt-get install -y php5-fpm php5-mysql php5-gd

COPY www.conf /etc/php5/fpm/pool.d/www.conf

# Clean up APT
RUN apt-get clean && rm -rf /var/lib/apt/lists/* /tmp/* /var/tmp/*

EXPOSE 9003
ENTRYPOINT /usr/sbin/php5-fpm --nodaemonize --fpm-config /etc/php5/fpm/php-fpm.conf

Build the php5-fpm image:

docker build -f Dockerfile.php -t "gankkank/ubuntu-trusty-php5-fpm:latest" .

Some hacking: nginx's default path is changed to /opt/site/apps and php5-fpm's socket path is changed to /opt/site/sock/php5-fpm-www.sock

Start a web container and an app container:

docker run -d -v /opt/site:/opt/site -t gankkank/ubuntu-trusty-php5-fpm:latest

docker run -d -p 8080:80 -v /opt/site:/opt/site -t gankkank/ubuntu-trusty-nginx:latest

Check the result by creating sample files at /opt/site/apps directory. <?php phpinfo(); ?> should return something.

Docker compose

Docker compose is provided as an alternative to Vagrant. It can start containers by defining them in a yaml file.

web:
  image: gankkank/ubuntu-trusty-nginx:latest
  ports:
    - "8000:80"
  volumes:
    - /opt/site:/opt/site
app:
  image: gankkank/ubuntu-trusty-php5-fpm:latest
  volumes:
    - /opt/site:/opt/site

Start all container as daemon: docker-compose up -d

If you want to use docker link between a web container and an app container, please change nginx config to use fastcgi_pass app:9003. And docker will add their hostname/ip records to /etc/hosts file in both containers.

web:
  image: gankkank/ubuntu-trusty-nginx:latest
  ports:
    - "8000:80"
  volumes:
    - /opt/site:/opt/site
app:
  image: gankkank/ubuntu-trusty-php5-fpm:latest
  volumes:
    - /opt/site:/opt/site

Another container may be needed to help us install php libraries and update app versions. Tools like composer will be useful. Many people also choose HHVM rather than PHP5-FPM. For more information please refer to the blog 1 to build a image by yourself.
