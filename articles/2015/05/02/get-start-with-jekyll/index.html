Get start with jekyll and github pages !!

May 2, 2015

# jekyll # github-pages

Oh, yeah, I decide to get start to write some technical posts with jekyll in github pages.

I would like to share with you how I get start and will update later when I find out some awesome tips!

Let's get started !

Install

You can install either github-pages gem or just jekyll gem.

Github-pages depends on jekyll and provides you several commands to help you work with Github Pages, such as github-pages health-check.

github-pages versions are also handy to check versions of installed gems.

Please check https://github.com/github/pages-gem for details.

Configure

Jekyll supports markdown and liquid, so it's good to learn some basic ideas about Github Markdown pages and Liquid wiki and also Jekyll.

Then create a git repository with jekyll new myblog, it will generate all the files needed.

It's important to change _config.yml. Add title, email, url, etc. In addition to some essential changes, I changed markdown to use redcarpet rather than kramdown to render markdown docs. kramdown is weired in some conditions such as when using ``` for the block content.

  markdown: redcarpet

Write and Build

Jekyll offers some handy commands to help preview your blog. jekyll serve --watch will auto generate new pages by watching the files in directories.

Default jekyll build will build your site at _site directory. You can change to other directory. For me, I just removed the _site directory and linked it to the directory gankank.github.io. After build, I can cd the directory to push new contents to github.

Deploy

I want to automate deploy steps when I decide to deploy new contents to github.

The general steps will be:

  • build new _site

  • add/commit new contents in _site

  • push them to my github repository gankank.github.io

So, I write my rake task to help me deploy contents to github. To deploy, run rake deploy, I default use the timestamp in the commit message when no commit message provides.

To deploy with a commit message, run rake deploy["my first commit"]

# Rakefile
require "rake"

task default: %w[deploy]

task :deploy, [:comment] do |t, args|
    # Build to _site
    `jekyll build`
    # Commit code in _site and push to github
    timestamp = Time.new.strftime("%Y-%m-%d %H:%M:%S").to_s
    args.with_defaults(comment: "auto build #{timestamp} HK")
    puts "deploy start"
    `cd _site; git add . --all; git commit -m "#{args[:comment]}"; git push`
end
